# Agent Instructions for Deadline Box

You are helping build Deadline Box, a personal deadline-based task submission system.

This is NOT a generic todo app.

Core product principle:
- Every task must become a deliverable.
- Every deliverable must have a DDL.
- Submission after DDL can never be marked as on-time.
- Missed tasks cannot be deleted.
- Late submissions can be accepted but must remain marked as late.
- All task status changes must be logged.

Day 1 scope:
Only implement the basic task submission loop.

Do NOT implement:
- real payments
- AI review
- weekly reports
- reward system
- wish account
- external reviewer mode
- complex auth
- file upload

Use clear, simple code.
Prefer working MVP over overengineering.
